# 安全策略

## 支持范围

安全修复优先覆盖当前最新正式版本和正在公开测试的候选版本。过旧版本可能只提供升级建议。

## 私下报告

请不要在公开 Issue 中直接发布可导致以下问题的完整利用细节：

- 开机失败、二屏卡死或系统服务崩溃；
- 任意命令执行、路径穿越或 ZIP 导入绕过；
- 未经预期的系统文件覆盖；
- WebUI 与 Root Shell 之间的命令注入；
- 诊断报告泄露敏感路径、账号或设备信息；
- 构建产物、更新文件或校验值被替换。

请通过 GitHub Security Advisory 的私密报告入口提交。报告中尽量包含：

- 受影响版本和提交；
- 设备、ROM、Android 版本和 Root 环境；
- 复现步骤；
- 预期行为与实际行为；
- 日志、截图或最小复现文件；
- 影响范围与建议修复方向。

请勿附带无授权商业字体。

## 响应原则

项目维护者会尽快确认问题、评估影响并准备修复。修复发布前，请避免公开会显著增加用户风险的细节。

## 使用风险

洛书属于 Root 模块，会修改系统字体挂载结果。请保留可进入 Root 管理器并禁用模块的恢复方式。候选版本应先在可恢复设备上测试。
