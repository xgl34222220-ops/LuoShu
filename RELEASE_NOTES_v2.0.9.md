# 洛书 v2.0.9 预发行版

本版本根治酷安标题与热度重叠、QQ 标签文字被裁掉一截的问题（v2.0.8 修复未触及真实原因的补完）。

## 主要更新

- 真根因修复：复合字体构建时 head 包围盒要在保存时才重算，归一化读到的 yMax/yMin 是中文基底的陈旧值，导入的英文/数字字形墨迹（1008/-323）从未被度量包住。现在归一化直接遍历全部字形轮廓（BoundsPen，glyf 与 CFF 通吃）取真实墨迹极值，hhea 与 typo 度量真正包住字形，includeFontPadding=false 的应用不再重叠、不再裁切。
- win 度量继续保持收敛上限：QQ 输入框等 includeFontPadding=true 场景不受影响。
- 负载架构升级 v6，归一化缓存升至 v5，复合与多字重缓存代际全量递增。
- 包含 v2.0.5-v2.0.8 全部修复：字体不再被静默重置；HyperOS 组合字体按真实分区镜像；应用成功清理重建重试计数。

## 测试提示

这是预发行版本。升级后请在洛书 App 内重新应用一次字体并完整重启。建议重点验证：酷安话题页标题与热度、QQ 联系人年龄标签与输入框、系统时钟、Play 商店搜索框，以及连续多次重启后的字体保持情况。反馈问题时请附设备系统、Root 方案、升级前版本、字体文件和完整复现步骤。
