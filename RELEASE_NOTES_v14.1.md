## 洛书 v14.1

### 完整中英数字体复合

- 以完整中文字体为基底，导入所选英文与数字字形
- 所有 ROM 字体槽使用同一份完整输出，不依赖缺字回退
- 英文或数字源字体即使包含中文，也不会抢占中文
- 支持 TrueType `glyf`、CFF、CFF2、TTC 与常见可变字体
- 修复 TrueType 复合保存阶段缺少 `xMin/yMin/xMax/yMax` 导致失败的问题

### 稳定性与性能

- 字体生成只由 WebUI 主动触发，刷写和开机阶段不处理大字体
- 真实阶段与进度显示，错误信息不再折叠为无意义代码
- 输出先在缓存区完整生成并验证，成功后事务提交
- 生成或 ROM 映射失败时自动保留原负载
- 开机可恢复被中断的事务
- 相同组合按 SHA-256 复用缓存，最多保留三份
- ColorOS `/data/fonts` 优先使用硬链接，减少大字体重复复制
- 精简内置运行时，降低正式包体积

### 安全边界

- 不覆盖 `fonts.xml` 或 `font_fallback.xml`
- 不裁掉中文、英文或数字字符
- 应用内置字体、网页网络字体、游戏资源字体不受模块控制

### 已验证

- Redmi K80 至尊版：HyperOS 3 / Android 16
- 一加 15：ColorOS 16
- TrueType、CFF、CFF2 与 TTC 复合构建
