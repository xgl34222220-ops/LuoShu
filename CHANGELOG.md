# 更新日志

本文件记录洛书的重要功能变化。正式发布版本以 GitHub Releases 中的标签、构建产物和 SHA-256 为准。

## [Unreleased]

### v14.3 Alpha1.7

- 新增 Material 3 Glass 与 Miuix 双皮肤架构，界面风格通过 DataStore 持久化并支持运行时即时切换。
- 新增跟随系统、浅色、深色三态，纯黑 AMOLED 模式、Monet 动态取色、种子色和柔和/鲜艳/中性 MaterialKolor 色板。
- 首页概览与界面设置页完成独立 Material / Miuix 两份 Compose 实现，只共用 UiState、Actions 和业务 ViewModel。
- Material 皮肤新增渐变高光状态卡和可关闭的 Haze 玻璃悬浮底栏；Miuix 皮肤保留超椭圆卡片、紧凑列表、大号状态和液态导航。
- 字体库、字体组合和日志本阶段继续使用共享业务层与过渡 UI，后续按首页模板逐页拆分。
- 系统全局粗细微调在两套首页中均完整保留。

### v14.3 Alpha1.6

- 原生 App 统一采用白泽当前的 Miuix 视觉系统，本版本只调整 UI，不新增字体功能或修改底层执行逻辑。
- 首页、字体库、字体工作台和日志页统一使用大标题、克制渐变、实心超椭圆卡片与紧凑分组列表。
- 底部导航改为图标在上、文字在下的液态悬浮栏，并增加滑动选中指示器和页面转场。
- Android 12 及以上接入系统动态色，深色模式与卡片层级同步白泽。
- “字重偏移”改为更易理解的“全局粗细微调”，隐藏不必要的技术术语。

### v14.3 Alpha1.5

- 区分字体原始默认、Android Regular 400 与当前设计轴参数。
- 同一字体且保持标准 Regular 400 时直接应用，不再执行无意义复合。
- 原生 App 接入系统全局粗细微调，并新增中文、英文、数字与标点覆盖诊断。

### v14.3 Alpha1.4

- Android 原生预览接入真实可变字体多轴参数。
- 静态多字重切换时加载对应真实字体文件。
- 完整设计轴参数提交到最终字体实例化任务。

### v14.3 Alpha1.3

- 多字重模块优先读取字体内部 `OS/2.usWeightClass`，文件名仅在内部元数据不可读时作为兜底。
- 使用内部 Family、Italic 与 Variable 信息归类字体；无字重文件名也可正确识别 100–900。
- 新增 ExtraLight 与 ExtraBold 两档，形成九档静态字重链路。
- 新增内部 200/800、文件名无字重提示的真实 ZIP 导入回归测试。

### v14.3 Alpha1.2

- 原生 App 预览不再依赖字体目录遍历顺序，稳定优先选择对应字体族的 Regular，其次 Variable，最后才回退其他档位。
- 预览桥返回实际源文件名和 SHA-256，便于确认不同字体模块是否最终导入了相同字体内容。
- 预览导出或 Android `Typeface` 加载失败时直接显示真实错误，不再静默回退系统默认字体造成“所有预览都一样”的假象。
- 可变轴读取与预览统一使用同一字体源。
- 新增原生预览源回归测试。

### v14.3 Alpha1.1

- 修复字体检测脚本被 `native_import.sh` 作为函数库加载时误执行 CLI，导致字体模块 ZIP 被当成普通字体校验并提前退出。
- 字体模块 ZIP 现在会正常进入只提取字体文件的安全导入器，不执行包内脚本。
- 导入桥保留底层错误并统一返回 JSON，不再出现“模块没有返回 JSON 数据”。
- 新增真实 ZIP 字体模块回归测试，验证模块元数据读取、安全解包、字体校验和复制结果。

### v14.3 Alpha1

- 原生 App 新增 Android 系统文件选择器，可一次选择多个 TTF、OTF、TTC 或字体模块 ZIP。
- 所选文件先复制到 App 私有缓存，再由 Root 桥验证受信任路径、文件大小、字体格式和安全类型。
- 字体导入按 SHA-256 去重；同名但内容不同的字体自动附加短哈希，不会静默覆盖。
- ZIP 继续使用只提取字体文件的安全导入器，不执行包内安装脚本或其他可执行内容。
- 导入结束后自动刷新字体库，并汇总成功、重复和失败项目。
- 新增基于完整 SHA-256 的稳定文件 ID，不再依赖可能重复或变化的显示名称。
- TTC/OTC 集合会枚举所有字体面，每个字体面拥有来源 SHA-256 与字体面索引组成的稳定 ID。
- 原生导入 TTC/OTC 时会安全拆分全部字体面，生成可独立预览、切换和参与组合的 TTF/OTF 文件。
- 拆分后的文件名包含内部 Family、Subfamily、字体面序号和来源短哈希，避免同名字体互相覆盖。
- 原生字体详情页显示内部名称、PostScript Name、真实字重、斜体、格式、字形数量、Unicode 覆盖和 CJK 数量。
- 详情页显示全部可变轴真实范围，并判断字体是否满足中文基底、英文或数字角色门禁。
- CI 真实生成双字体面 TTC，验证拆分数量、独立字体可读性和稳定身份数据。

### v14.2 RC2.1

- 修复字体组合角色门禁把半角标点误设为必需项的问题。
- 中文基底只要求必要中文、大小写英文字母和数字槽位；英文与数字角色只检查真正必需字形。
- 花轮丸等上一版本可正常使用、但缺少部分可选标点的字体不再被误拦截。
- 版本号提升至 14221，方便确认刷入和完整重启后新模块确实生效。

### v14.2 RC2

- 原生 App 启动后可重新接管正在运行的字体切换和复合字体任务。
- 模块持久返回任务 ID、任务类型、真实进度和待重启状态，关闭 App 后状态不再丢失。
- 多轴任务统一使用版本无关的状态、配置与工作进程文件名，升级时清理实验版本残留。
- 可变字体提交范围放宽到字体规范允许的 1–1000，并由字体实例化器按各字体真实轴范围最终校正。
- App 状态优先显示正在运行的任务；多个历史任务并存时显示最近完成的一项。
- 中文、英文和数字组合角色在任务入队前分别检查关键字形覆盖。
- 安装脚本缩减为安全部署流程，不在刷写阶段生成字体、结束进程或直接安装 App。
- 新增精简终端入口，只提供状态、列表、应用、恢复默认和日志。
- WebUI 隐藏重启 SystemUI 的实验入口，所有字体变更统一要求完整重启。
- Validate 工作流开始编译并测试原生 App、角色门禁与最终模块包，正式签名发布前强制运行完整检查。
- 发布说明根据当前版本自动选择，APK 发布前验证签名方案和证书摘要。

### v14.2 RC1

- 版本推进至 v14.2 RC1，并新增原生 MIUIx 风格管理 App。
- 同一工作流同时生成独立 APK 与内置 App 的模块 ZIP。
- 模块刷入时尝试安装 App，并提供模块“操作”按钮作为安装回退。
- 原生字体预览移至 IO 线程并限制缓存容量。
- 统一以 `module.prop` 驱动模块、App、WebUI 和产物版本。
- PR 提供 Full/Lite 测试包；正式 Release 只接受固定密钥签名并禁止覆盖 Tag。
- 全局直替字体增加中文、英文、数字和标点覆盖门禁，缺字时提示改用复合字体。
- 缓存 ARM64 复合字体运行时，减少重复 CI 构建时间。
- 将洛书当前源码许可证由 MIT 调整为 GPL-3.0-only；历史发行版本继续适用其原许可证。
- 统一项目说明、模块说明、包内 README 和许可证声明。
- 修复构建脚本错误选择 CPython 许可证的问题。
- 增加第三方组件声明、安全报告与贡献规范。
- 彻底移除 Emoji、图标字体替换的公开功能与遗留执行路径。
- 保留图标字体和彩色字体识别，仅用于安全过滤。
- 改进复合任务真实进度、缓存状态和事务恢复。

## [v14.1] - 2026-07-17

- 引入完整复合字体引擎。
- 以中文字体为完整基底，导入英文和数字字形及度量。
- 统一系统文字物理槽，避免英文或数字源字体自带中文时抢占中文。
- 支持 glyf、CFF、CFF2、TTC 和常见可变字体。
- 修复 glyf 字形边界缺失导致的 `xMin` 保存失败。
- 不覆盖 `fonts.xml` 或 `font_fallback.xml`。
- 刷写和开机阶段不生成大字体。
- 增加 SHA-256 缓存、输出验证、事务提交和失败回滚。
- 适配 ColorOS 16、HyperOS 3、Android 16 与常见 Root 管理器。

## [v14] - 2026-07-17

- 完善 Android 16、ColorOS 16 和 HyperOS 3 字体路径适配。
- 增加 Google 应用、微信 XWeb、ROM 字体槽和 Mountify 相关处理。
- 改进 WebUI、字体扫描、导入和诊断能力。
