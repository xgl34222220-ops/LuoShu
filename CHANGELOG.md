# 更新日志

本文件记录洛书的重要功能变化。正式发布版本以 GitHub Releases 中的标签、构建产物和 SHA-256 为准。

## [Unreleased]

- 统一项目说明、模块说明、包内 README 和许可证声明。
- 修复构建脚本错误选择 CPython 许可证的问题。
- 增加第三方组件声明、安全报告与贡献规范。
- 彻底移除 Emoji、图标字体替换的公开功能与遗留执行路径。
- 保留图标字体和彩色字体识别，仅用于安全过滤。
- 改进复合任务真实进度、缓存状态和事务恢复。

## [v14.1] - 2026-07-17

- 引入完整复合字体引擎。
- 以中文字体为完整基底，导入英文和数字字形及度量。
- 统一系统文字物理槽，避免英文或数字源字体自带中文时抢占中文。
- 支持 glyf、CFF、CFF2、TTC 和常见可变字体。
- 修复 glyf 字形边界缺失导致的 `xMin` 保存失败。
- 不覆盖 `fonts.xml` 或 `font_fallback.xml`。
- 刷写和开机阶段不生成大字体。
- 增加 SHA-256 缓存、输出验证、事务提交和失败回滚。
- 适配 ColorOS 16、HyperOS 3、Android 16 与常见 Root 管理器。

## [v14] - 2026-07-17

- 完善 Android 16、ColorOS 16 和 HyperOS 3 字体路径适配。
- 增加 Google 应用、微信 XWeb、ROM 字体槽和 Mountify 相关处理。
- 改进 WebUI、字体扫描、导入和诊断能力。

早期版本的详细历史可在 Git 提交与 Releases 中查看。
