# 洛书 v2.0.2

本版本修复 v2.0.1 修复包没有真正替换用户当前正式 App、且未即时解除当前模块 `disable` 的发布与迁移问题。

- 使用仓库固定签名构建正式 `io.github.xgl34222220.luoshu` APK，能够覆盖安装已有正式洛书；不再把并排安装的 Debug App 当成用户修复包。
- 模块刷入阶段同时处理待安装目录与当前 `/data/adb/modules/LuoShu`，当场删除旧安全守卫误设的 `disable` 及其失败状态。
- 用户明确执行“应用字体”或“恢复系统字体”时视为重新启用洛书，即使旧更新已丢失失败计数，也会解除遗留 `disable`。
- `remove` 仍作为 Root 管理器的明确卸载请求保留，不会被字体操作静默撤销。
- 完整保留 v2.0.1 的 Magic Mount RC/RS 分区补齐、HyperOS GoogleSans/Roboto 槽位覆盖、统一中英数字垂直度量、Mono 派生与统一预览修复。
- 版本提升至 `v2.0.2 / 20002`，正式 App `versionCode` 为 `2000201`。

刷入正式模块后重启一次，再从原来的洛书图标进入。若当前字体负载来自旧度量缓存，首次开机会在后台重建，完成后按 App 提示再重启一次。
