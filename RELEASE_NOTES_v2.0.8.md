# 洛书 v2.0.8 预发行版

本版本根治酷安标题与热度重叠、QQ 标签文字被裁掉一截的问题。

## 主要更新

- 根因修复：字体归一化设置了 fsSelection USE_TYPO_METRICS 标志，渲染引擎按 OS/2 typo 度量排版；此前 typo 保持紧凑契约，导致 v2.0.6 的"hhea 包住字形轮廓"被架空。现在 typo 与 hhea 一致包住字形真实轮廓（下限仍为 0.928/0.244 原厂契约），includeFontPadding=false 的应用不再重叠、不再裁切。
- win 度量保持收敛上限不变：QQ 输入框等 includeFontPadding=true 的场景完全不受影响，不会重新引入文字抬高。
- 负载架构升级 v5：升级后首次开机自动后台重建；应用字体成功时同步清理重建重试计数。
- 归一化缓存升至 v4，复合与多字重缓存代际全量递增，杜绝旧度量缓存复用。
- 包含 v2.0.5-v2.0.7 全部修复：字体不再被静默重置；HyperOS 组合字体按真实分区镜像，数字英文全局生效。

## 测试提示

这是预发行版本。升级后请在洛书 App 内重新应用一次字体并完整重启。建议重点验证：酷安话题页标题与热度、QQ 联系人年龄标签与输入框、系统时钟、Play 商店搜索框，以及连续多次重启后的字体保持情况。反馈问题时请附设备系统、Root 方案、升级前版本、字体文件和完整复现步骤。
