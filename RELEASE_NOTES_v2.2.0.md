# 洛书 v2.2.0

这是洛书 v2.2.0 正式版，重点解决不同 ROM 字体结构差异、组合字体生成速度以及组合页排版回归。

## 主要更新

- **设备自适应原厂字体清单**：revision 3 扫描器按当前设备读取系统分区、OEM 字体目录与字体 XML，分别记录原厂字体路径数、物理唯一文件数和安全可替换 UI 槽位。主题字体、活动覆盖与字体挂载单独诊断，不再混入原厂槽位统计。
- **恢复快速组合**：静态多字重字体默认只生成当前选择的字重，不再无提示扩展为 100–900 九档大型 CJK 复合；真正带 `wght` 设计轴的可变字体仍支持完整自动多字重生成。
- **修复组合页竖排问题**：长字重能力标签改为紧凑的档位数量摘要，避免把“英文字形”等标题和说明压缩成单字宽竖排。
- **后台任务与安全性**：保留独立 Root Worker、任务状态恢复、字体切换回滚、缓存复用和重启生效保护，降低任务中断或切换失败造成的风险。
- **正式原生 App**：模块继续内置与独立安装包字节一致的固定签名 App，支持覆盖安装；正式产物同时提供 SHA-256 校验文件。

## ROM 兼容说明

洛书使用同一套自适应引擎，但不会给所有 ROM 强行套用相同的字体路径或固定槽位数量。当前重点验证环境为 HyperOS 3 与 ColorOS 16；其他 ROM 会根据设备实际分区、XML、字体文件和 OEM 标记生成独立清单。

例如两台设备都显示 19 个槽位，只表示它们各自筛选出了 19 个安全可替换 UI 文件槽位，不代表两台设备使用同一份映射。

## 安装与升级

刷入 `LuoShu-v2.2.0.zip` 后完整重启。模块会优先覆盖更新内置 App；也可以单独安装 `LuoShu-App-v2.2.0.apk`。从旧版本升级前建议保留当前可用模块包，首次使用新组合字体时先完成一次完整重启。
