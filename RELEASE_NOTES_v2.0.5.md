# 洛书 v2.0.5 预发行版

本版本修复一个升级后可能反复把字体静默重置回系统默认的严重问题：应用新字体成功后，重启仍会被误判为“旧负载重建失败”，进而删除用户正在使用的字体并回退默认，形成每次重启都丢失字体的死循环。

## 主要更新

- 修复升级后“待重建”标记未在字体应用成功时清除的问题，避免重启后对已是新架构的负载重复重建。
- 调整开机服务顺序：字体索引预热先于旧负载重建执行，杜绝索引未就绪导致重建在一秒内误报失败。
- 重建失败不再立即隔离删除当前字体，改为保留用户正在使用的字体并在下次开机自动重试，仅连续 3 次失败才安全回退系统默认字体，并通过通知明确提示。
- 重建成功后同步清理重试计数，状态机更健壮。

## 测试提示

这是预发行版本。由 v2.0.4 或更早版本升级后，请务必在洛书 App 内重新应用一次字体并完整重启，重启后确认字体不再回退为系统默认。建议同时验证 mix 复合字体、单字体切换、Play 商店搜索框字体跟随、以及连续多次重启后的字体保持情况。反馈问题时请附设备系统、Root 方案、升级前版本、字体文件和完整复现步骤。
