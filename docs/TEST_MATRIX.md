# 洛书发布测试矩阵

本文件记录自动化门禁和已经完成的真机验证进度，**不是 ROM 或机型支持白名单**。

洛书安装时会在每台设备上扫描真实字体目录和字体配置，生成设备独立的原厂字体清单；运行时优先按照该清单映射系统 UI 字体。未列在下方真机表格中的设备仍会执行完整扫描和清单应用，只是尚未由项目维护者完成整套页面、回滚、任务恢复和特殊字体机制验证。

## 设备自适应机制门禁

- 扫描 `system`、`system_ext`、`product`、`my_product`、`vendor` 的字体目录与配置；
- 扫描 `odm`、`oem`、`my_region`、`hw_product` 作为扩展诊断来源；
- 解析 `fonts.xml`、`font_fallback.xml`、`fonts_customization.xml` 等实际存在的配置；
- 记录真实路径、分区、family、alias、字重、样式、TTC index、UPEM、`hhea` 与 `OS/2` 度量；
- 使用设备号和 inode 去重，不把分区别名或硬链接重复统计成独立字体；
- 主题字体和活动字体挂载只记为覆盖证据，不混入原厂清单；
- 应用字体时必须优先使用有效的设备原厂清单；
- 清单缺失、过期、损坏或没有可映射槽位时，才允许进入厂商静态适配或最小 AOSP 回退；
- OTA 或系统指纹变化后必须使旧清单失效并重新扫描。

## 自动化门禁

- Shell、Python 语法检查通过。
- 原生 App Kotlin 编译与单元测试通过。
- 唯一模块 ZIP 可构建，ZIP 完整性与 SHA-256 校验通过。
- 模块内 APK 与独立 APK 字节一致，缺失内置 App 时构建必须失败。
- 模块不得包含 `webroot/`，`module.prop` 不得声明 `webroot=`。
- 不得生成或上传 Lite、App-less 或其他无内置 App 的模块变体。
- 全局字体必须通过中文、英文、数字和标点覆盖率门禁。
- 组合任务入队前必须分别通过中文基底、英文字体和数字字体角色门禁。
- Latin-only 测试字体必须被中文基底门禁拒绝。
- 模块不得包含 `skip_mount` 或 `skip_mountify`。
- 元模块镜像只在字体事务成功后同步，不在开机脚本注入全量同步。
- 安装脚本不得在刷写阶段直接安装 App、结束系统进程或生成大字体。
- 安装后的 `洛书` 命令不得暴露上一字体回滚、字体缓存热刷新或重启 SystemUI。
- 正式发布前必须执行完整源码检查、App lint/测试、APK 签名校验与模块成品检查。

## App-only 单包专项回归

- Root 管理器中不再出现模块 WebUI 入口。
- 模块包始终内置 App，可通过模块“操作”按钮安装或更新。
- App 在复合任务运行中被划掉或系统回收，重新打开后能继续显示同一任务 ID 和真实进度。
- App 在字体任务完成后关闭再打开，仍显示待完整重启状态。
- 同时存在历史切换任务和复合任务时，优先显示正在运行的任务；均已结束时显示最近完成的一项。
- 测试真实 `wght` 范围不是 100–900 的可变字体，App 显示范围与最终实例化结果一致。
- 中文基底缺少常用中文、英文字母、数字或标点时，在任务入队前直接拒绝。
- 英文字体缺少 A–Z/a–z 或数字字体缺少 0–9 时，在任务入队前直接拒绝。
- 模块内 App 不在刷写阶段强制安装；重启后通过模块“操作”按钮可安装或更新。

## 任务中心专项回归

- 字体扫描、导入、应用、恢复、组合、删除与重启记录能被正确分类。
- 进行中任务显示真实进度；无百分比的短任务不得伪造进度。
- 失败任务使用错误状态，不得被包含“完成”等片段的文案误判为成功。
- 字体准备完成后显示“等待重启”，完整重启后该状态应消失。
- 当前后台任务与历史日志重复时只显示一条，当前状态优先。
- 退出任务中心再返回，当前任务与最近历史仍然存在。
- 原始日志可滚动、长按选择和复制，不影响任务时间线滚动。
- Material 与 Miuix 两种界面显示相同的任务数量、状态和进度。

## 发布规则

- Alpha/Beta/RC 只发布为 GitHub prerelease。
- 正式版必须使用固定签名，并保留同一签名的历史 APK。
- 标签只创建一次；发布流程禁止移动或覆盖已有标签。
- 任一系统出现黑屏、SystemUI 重启、批量闪退或方框乱码，立即停止发布并恢复上一可用模块包。
