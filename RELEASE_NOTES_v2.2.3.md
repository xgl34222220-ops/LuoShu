# 洛书 v2.2.3

洛书 v2.2.3 是针对 v2.2.2 正式版的稳定性修复版本，重点解决设备字体状态长期无法验证、应用字体失败或超时，以及任务中心操作按钮未对齐的问题。

## 主要修复

- 完整重启后自动执行设备字体加载验证，不再要求用户反复应用字体或手动检查。
- 等待 Android 启动完成并确认字体事务空闲后再验证，避免开机阶段过早读取造成永久“尚未验证”。
- 对部分 HyperOS、ColorOS 等不公开完整 FontManager 字体明细的系统，在全部系统可见挂载文件与洛书负载 SHA-256 一致时，显示明确的“挂载已验证”状态。
- 每次应用新字体后重新进入“等待完整重启验证”，不会沿用上一套字体的旧验证结果。
- 应用字体前台阶段改为快速文件与格式预检，完整字形、覆盖与安全验证继续在后台切换事务中执行。
- 字体切换任务改为独立 Root Worker；切到后台、关闭 App 或 App 被系统回收后，任务仍可继续。
- 安全超时由 45 秒调整为 110 秒，适配大型 CJK、TTC/OTC 字体和较慢存储设备；真正超时仍会终止任务并回滚上一套字体。
- 修复 MIUIx 任务中心右上角诊断与刷新按钮视觉中线不一致。
- 首页未验证提示改为“完整重启后将自动检查，无需重复应用”，并支持显示“设备字体挂载已验证”。

## 安装与升级

1. 在 Magisk、KernelSU、SukiSU Ultra 或 APatch 中刷入 `LuoShu-v2.2.3.zip`。
2. 完整重启设备；本次版本会在开机完成后自动验证当前字体的实际加载状态。
3. 打开洛书并授予 Root 权限。

模块 ZIP 内置同版本正式签名 App。独立的 `LuoShu-App-v2.2.3.apk` 可用于单独安装或覆盖升级。

## 版本信息

- 模块版本：`v2.2.3`
- 模块 versionCode：`20203`
- App versionCode：`2020301`
- 最低 Android：Android 9 / API 28
- 目标 Android：API 36

## 校验

Release 同时提供模块 ZIP、正式 APK 及各自的 SHA-256 文件。下载后可使用对应 `.sha256` 文件核验完整性。
