# 洛书 v2.0.7 预发行版

本版本修复 HyperOS 设备使用组合字体时"中文已更换、数字与英文仍显示原厂字体"的问题。

## 主要更新

- HyperOS 组合字体负载按真实分区镜像：数字字重（100-900.ttf）与拉丁/核心 MiSans 文件在部分机型只存在于 /product 或 /system_ext 分区，此前 mix 负载只写入 system/fonts，无法覆盖真实文件。现在 mix 应用后与 ColorOS 一样同步到真实分区，中文、英文、数字全局统一生效。
- 包含 v2.0.6 全部修复：hhea 度量包住字形轮廓极值，修复 includeFontPadding=false 应用的标题重叠与标签裁切；负载架构 v4，升级后自动后台重建。
- 包含 v2.0.5 全部修复：升级后字体不再被静默重置回默认。

## 测试提示

这是预发行版本。升级后请在洛书 App 内重新应用一次组合字体并完整重启。建议重点验证：系统时钟数字与英文、酷安标题与热度、QQ 年龄标签与输入框、Play 商店搜索框，以及连续多次重启后的字体保持情况。反馈问题时请附设备系统、Root 方案、升级前版本、字体文件和完整复现步骤。
