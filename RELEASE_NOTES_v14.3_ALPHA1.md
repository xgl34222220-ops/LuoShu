# 洛书 v14.3 Alpha1.2

v14.3 Alpha1.2 修复原生字体预览绑定和静默回退，并保留 Alpha1.1 的字体模块 ZIP 导入修复。

## Alpha1.2 修复

- 原生 App 预览不再按目录遍历顺序随便取同字体族的第一个文件；优先稳定选择 Regular，其次 Variable，最后才使用其他可用档位。
- 预览桥返回实际源文件名与 SHA-256，方便判断两个字体模块最终是否导入了同一份字体内容。
- 预览导出、文件为空或 Android `Typeface` 加载失败时，界面直接显示真实错误，不再悄悄退回系统默认字体造成“所有预览都一样”的假象。
- 可变轴读取与预览使用同一份确定的字体源，避免预览和字重范围来自不同文件。
- 新增预览源回归测试，验证多字重字体族始终选择 Regular，并验证缺少 Regular 时的安全回退。

## Alpha1.1 修复

- 修复 `font_check.sh` 被导入桥作为函数库加载时误执行命令行入口的问题。
- 修复 TTF、OTF、TTC 可以导入，但字体模块 ZIP 在进入安全解包器前被当成普通字体校验并提前终止的问题。
- 字体模块 ZIP 现在会正常进入只提取字体文件的安全导入器，不执行包内脚本。
- ZIP 导入失败会保留底层错误并统一返回 JSON，不再只显示“模块没有返回 JSON 数据”。
- 新增真实字体模块 ZIP 回归测试，验证模块元数据读取、安全解包、字体校验与复制结果。

## Alpha1 功能

- 可一次选择多个 TTF、OTF、TTC 或字体模块 ZIP。
- 所选文件先进入洛书 App 私有缓存，再由 Root 桥读取。
- Root 桥只接受洛书正式版或 Debug App 的受信任缓存路径。
- 单个文件限制为 256 MB，字体会检查真实格式并拦截无效或不安全字体。
- 字体按 SHA-256 去重；同名但内容不同的文件使用短哈希区分，不覆盖已有字体。
- 导入完成后自动刷新字体库，并分别报告成功、重复和失败数量。
- 新增稳定文件 ID `sha256:<哈希>`，不再把可变的显示名称当作唯一身份。
- TTC/OTC 集合会枚举全部字体面，每个字体面拥有 `sha256:<哈希>:face:<索引>` 来源身份。
- 通过原生导入选择 TTC/OTC 时，会将每个字体面安全拆成独立 TTF/OTF 文件；拆分后的字体直接进入字体库，可独立预览、切换和参与中文/英文/数字组合。
- 拆分文件名包含内部 Family、Subfamily、字体面序号和来源短哈希，不会覆盖同名字体。
- 原生字体详情可查看内部 Family、Subfamily、Full Name、PostScript Name、格式、真实字重、斜体状态、字形与 Unicode 数量。
- 详情页显示全部可变轴的最小值、默认值和最大值，并判断是否适合作为中文基底、英文或数字字体。
- CI 会真实生成双字体面 TTC，验证拆分数量、独立字体可读性和稳定身份数据。

本版本为开发测试版，不建议作为最终稳定版本长期使用。
