# 洛书 v14.3 Alpha1.11

## 原生字体库性能重构

本版本不增加字体功能，重点让原生 App 真正做到流畅和字体库秒开。

## 缓存优先加载

- App 私有目录新增原子字体索引，持久保存字体名称、格式、字重、可变状态、有效性、大小与日期。
- App 启动后立即在 IO 线程读取本地索引；进入字体库时优先显示已有数据，不再等待 Root 全量扫描。
- App 更新使用覆盖安装，索引、字体预览缓存与 Material / Miuix 外观设置都会保留。
- 后台刷新失败只显示提示，不会清空已经加载的字体列表。

## 轻量目录指纹

- 新增 `common/font_library_cache.sh`，只读取文件名、大小与修改时间，不解析字体轮廓。
- 字体目录指纹未变化时完全跳过字体列表解析。
- 新增、修改、删除、导入字体或用户主动点击刷新时才重建完整索引。
- 删除字体成功后先从 App 列表和本地索引移除，再后台校准，不再等待整库扫描后才更新界面。
- 新增回归测试，验证指纹稳定以及新增、修改、删除时正确失效。

## 开机后台预热

- `service.sh` 在系统启动完成后比较轻量指纹。
- 指纹未变化时不执行任何字体分析。
- 指纹变化或模块 JSON 缓存缺失时，才在后台重建模块字体列表。
- 因此首次打开 Alpha1.11 App 时也可优先命中模块预热结果。

## 字体预览优化

- 增加进程内 Typeface LRU 缓存，页面切换和列表回收后不重复创建同一字体对象。
- 同一字体的并发预览请求会合并，不再重复启动多个 `su` 导出任务。
- Root 预览导出最多同时执行两个，避免快速滚动时大量进程争抢 IO。
- 持久预览缓存由 12 个文件提升到最多 32 个，并设置 384 MB 总量门禁。
- 可变字体轴元数据增加进程内缓存，反复进入组合页不再重复分析同一字体。

## Full 模块自动更新 App

- Full 模块继续内置 App，并在刷写阶段或首次开机自动覆盖更新。
- 覆盖更新保留本地字体索引和预览缓存。
- Lite 包继续不内置 App。

## 保持不变

- Material 3 Glass / Miuix 双皮肤架构与外观设置。
- 真实可变字体全部 `fvar` 轴预览和最终提交。
- 静态多字重真实源文件切换。
- 同字体标准 Regular 400 直接应用。
- 系统全局粗细微调。
- 中文、英文、数字与标点覆盖诊断。
- TTF、OTF、TTC 和字体模块 ZIP 安全导入。

本版本继续作为 Draft PR 真机性能测试版本，不合并、不发布。
