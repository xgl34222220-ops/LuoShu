# 洛书 v14.3 Alpha1.5

## 字重语义分层

- 可变字体工作台不再把所有“默认值”混为一谈：界面分别显示字体 `fvar` 原始默认、Android 语义上的 Regular 400，以及用户当前选择的轴参数。
- `wght` 轴提供“跟随字体原始默认”和“Regular 400”快捷选择，便于对比字体作者设计默认与系统常规字重。
- 系统全局粗细微调已接入原生 App，使用 Android 字重偏移接口在 300–700 安全范围内即时写入；未更新的应用通常只需重新打开。

## 同字体直应用

- 中文、英文、数字三个槽位选择同一字体，且均保持标准 Regular 400、没有额外设计轴时，工作台自动切换为“直接应用此字体”。
- 同一字体但存在自定义 `wght`、`wdth`、`opsz` 等参数时仍走实例化和复合生成，避免预览与最终效果不一致。

## 字形覆盖诊断

- 工作台可检查字体的中文、基本拉丁字母、数字和常用标点覆盖率，并给出当前槽位的 fallback 风险提示。
- 覆盖诊断用于定位字体本身缺字导致的字体不一致；应用内置字体、WebFont 或包级字体资源仍需单独诊断。

## 导入与实时预览

- TTF、OTF、TTC 继续作为推荐的直接导入方式；字体模块 ZIP 保留为高级导入入口，不要求普通用户手动寻找模块内字体文件。
- 保留 Alpha1.4 的 Android `Typeface.Builder.setFontVariationSettings()` 真实多轴实时预览、静态多字重真实文件切换，以及完整轴参数提交。
- 字体设计轴拖动只更新 App 内预览；点击应用后才写入模块。系统字体文件变更仍建议完整重启。

本版本为开发测试版，PR 继续保持 Draft，不合并、不发布，等待真机验证。
