# 洛书 v14.3 Alpha1.9

## 双皮肤架构第三阶段

本阶段不增加字体功能，重点完成双皮肤 UI 的收尾、旧界面删除和外观状态回归测试。

## 正式运行入口

- App 入口收敛为 `LuoShuAppShell`。
- 删除 Alpha1.5、Alpha1.6、旧单皮肤 App、共享双皮肤过渡页、Stage2 临时壳和旧底栏作用域文件。
- 首页、字体库、字体组合、日志和设置继续根据 `UiStyle.MATERIAL / UiStyle.MIUIX` 进入各自页面实现。
- 字体业务 ViewModel、Root 桥、真实字体预览和复合生成逻辑保持单份。

## 全局弹层换皮

- 应用字体、删除字体和恢复系统字体确认弹窗分别实现 Material 与 Miuix 样式。
- 组合字体选择器分别实现 Material 与 Miuix 样式，并加入弹窗内本地搜索。
- 字体详情选择、详情内容、导入结果和分析进度均进入双皮肤主题。
- Material 弹层使用标准 M3 shape、按钮和状态容器。
- Miuix 弹层使用大标题、紧凑信息层级、超椭圆面板和白泽式操作按钮。

## 安全导入浮层

- 导入浮层移入 `LuoShuTheme` 内部，正确继承 MaterialKolor、Monet、深色和纯黑设置。
- Material 使用圆形详情按钮与标准扩展浮动操作。
- Miuix 使用超椭圆详情按钮和饱满胶囊导入操作。
- 导入入口只在字体库与字体组合页面显示，减少首页、日志和设置页遮挡。
- TTF、OTF、TTC、模块 ZIP、安全缓存、256 MB 限制、SHA 去重和 Root 导入逻辑未修改。

## 外观状态稳定性

- DataStore 读取后统一调用 `AppearanceSettings.normalized()`。
- 非法种子色自动回退到洛书蓝。
- 玻璃关闭时，旧数据中残留的模糊开关会自动关闭。
- 开启模糊时会再次检查玻璃是否启用，避免保存无效组合。
- 新增 JVM 测试覆盖两种界面风格、三态深色模式、旧取色值兼容、种子色回退和玻璃/模糊约束。

## 保留能力

- 真实可变字体全部 `fvar` 轴实时预览与最终提交。
- 静态多字重真实源文件切换。
- 同字体标准 Regular 400 直接应用。
- 系统全局粗细微调。
- 中文、英文、数字与标点覆盖诊断。
- 后台任务恢复、完整重启提示与失败回滚。

本版本为双皮肤架构清理后的真机测试版。PR 继续保持 Draft，不合并、不发布。
